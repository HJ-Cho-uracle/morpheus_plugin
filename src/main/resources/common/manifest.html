Overview

  1. Provides Manifest.xml for application basic configuration.
  2. The platform automatically reads and uses the information in Manifest.xml when the application runs.
  3. Location of Manifest.xml : Resource(res) Folder structure Reference



Precautions when distributing apps and resources

  1. Be careful that Manifest.xml is not included in the resource distribution file
    because it uses files from within the app (asset (Android) or bundle (iOS)).
  2. <resource><target> Set it to 'doc'.
  3. <resource><update><mode> Set to 'real'.
  4. <log> Set to 'n'.



Logging <log>

  • Sets whether to enable logging.
  • enable : 'y' / disable : 'n'

Multilingual setup <lang>

  • Sets the language of the application.
  • When set to 'system', the language set in the device is available.
  • Language - Define folder names by country
    • The language is ISO complies with 639, BCP 47 (Reference)
    • The country ISO is defined in 3166 (Reference)

Resource <resource>

  • You can make web resource related settings.
  • Example Code
        <!-- resource setting -->
    <resource>
        <base_version>100001</base_version> <!-- base resource version -->
        <target>app</target>  <!-- use resource target / app, doc, sdcard -->
        <update>
            <!-- resource update check transaction info -->
            <server>HTTP_REST</server> <!-- resource update target server  -->
            <trcode>/api/res/auto-update</trcode> <!-- /rest/api/res/auto-update(rest) , rsc.auto_update(normal) -->
            <mode>real</mode> <!-- resource update mode / dev, real -->
        </update>
    </resource>

Resource base version <base_version>

  • It is a version of the resource that goes up to the market, and when the user downloads the app from the market and executes it, the update from the basic version to the latest version can be received.
  • Example Code
        <!-- Resource base version -->
    <base_version>100001</base_version>

Used resource targets <target>

  • Set the location of the web resource you want to use.
  • Type
    1. app : Use the resources contained within the app (used if you do not use development and resource updates).
    2. doc : Use resources in the space available for read / write access to the app (used when deploying using resource updates and resource copying).
    3. sdcard : Use the resources contained in the sdcard area (used in development). - Support OS : Android
  • Example Code
        <!-- Resource Target -->
    <target>app</target>

Resource Update <update>

  • Make settings related to resource update.
Resource Update Server Target <server>
  • <network> > <http> You must set the gateway server name with the resource update function to the server target set in
  • Example Code
        <!-- Gateway server target name -->
    <server>HTTP_REST</server>
Resource update service code <trcode>
  • Set the resource update service code.
  • There are two types of update servers depending on the type of service support (rest or interface.do).
  • Rest system
    • '/api/res/auto-update'
    • <network><http><'TARGET_SERVER_NAME'><address> and adding the trcode to the URL.
  • interface.do system
    • 'rsc.auto_update'
  • Example Code
        <!-- Resource update service code -->
    <trcode>/api/res/auto-update</trcode>
Resource Update Service Mode <mode>
  • Sets the resource update service mode.
  • Choose which resources the update server will have, called 'development deployment (dev)' or 'operational deployment (real)'.
  • development deployment : dev / operational deployment : real
  • Example Code
        <!-- Resource update service code -->
    <mode>real</mode>

First run page <startpage>

  • Sets the initial execution page.
  • Example Code
        <!-- First Initialization Page -->    
    <startpage>
        <name>index.html</name>
        <orient>port</orient>
    </startpage>

Start page name <name>

  • Set the start page.
  • In the initial execution page, the server and resource dynamic update must be performed.

Start page orientation <orient>

  • Set the orientation of the start screen.
  • The type of orient mode.
    1. land : Landscape mode
    2. port : Portrait mode
    3. all : Landscape and portrait mode
    4. default : Apply the mode set by the system

Start page orientation Tablet <orient>

  • Set the orientation of the start screen.
  • Types of orient mode.
    1. land : Landscape mode
    2. port : Portrait mode
    3. all : Landscape and portrait mode
    4. default : Apply the mode set by the system

Full screen default setting <default_action>

  • Sets the default mode for orient, animation, and indicator applied to the entire screen.
  • Example Code
        <default_action>
        <orient>default</orient> <!-- default orientation -->
        <animate>default</animate> <!-- default animation -->
        <indicator>y</indicator> <!-- show page switching indicator -->
    </default_action>

Set full screen default direction <orient>

  • Sets the default orientation applied to the entire screen.
  • default orient mode의 종류.
    1. land : Landscape mode
    2. port : Portrait mode
    3. all : Landscape and portrait mode
    4. default : The mode set by the system or app.

Set full-screen default animation <animate>

  • Sets animate between default screens applied to all screens.
  • Scheduled
  • Default animate The type of mode.
    1. default : System default animation.
    2. none : No animation
    3. slide_left : PUSH to the left
    4. slide_right : PUSH toward today
    5. slide_top : Upward PUSH
    6. slide_bottom : PUSH down
    7. zoom_in : Zoom in
    8. zoom_out : Zoom out
    9. fade : Fade
    10. modal_left : Moving while overlapping left
    11. modal_right : Moving with overlapping to the right
    12. modal_up : Moving upwards overlapping
    13. modal_right : Moving while overlapping downwards

Set whether or not to display indicators when moving the entire screen <indicator>

  • Sets whether to display the indicator when moving the entire screen.
  • enable : 'y' / disable : 'n'

Library Settings : <library>

  • Set up the platform provided libraries (addon, plugin) and extension interface.
  • On iOS, only the last class name is used.
  • Example Code
        <library>
        <ext> m.client.app.implementation.ExtendWNInterface </ext>
        <addons>
            <path>m.client.library.addon.db.WNInterfaceDB</path>
            <path>m.client.library.addon.file.WNInterfaceFile</path>
            <path>m.client.library.addon.media.WNInterfaceMedia</path>
            <path>m.client.library.addon.net.WNInterfaceNet</path>
            <path>m.client.library.addon.netext.WNInterfaceNetExt</path>
            <path>m.client.library.addon.popup.WNInterfacePopup</path>
            <path>m.client.library.addon.zip.WNInterfaceZip</path>
        </addons>
        <plugins>
            <path>com.google.zxing.client.android.WNInterfaceQR</path>
        </plugins>
    </library>
 

Expansion interface <ext>

  • Enter the full package name of the native extension interface class.

Add-on <addons>

  • Addons that provide frequently used functions by default.
  • It is recommended not to modify it unless it is a special situation.

Plugin <plugins>

  • Enter the extension interface class full package name of the plugin libraries provided by the plugin.
  • The full package name is in the plug-in's description file.

Network settings : <network>

  • Set up http and socket networks.

Http setting <http>

  • Create a network alias by giving it an http server name.
  • Example Code
    <HTTP_REST> 
    <path>m.client.app.samples.network.HttpDataNetworkManager</path>
    <address>http://61.74.156.194:83/rest/</address>
    <timeout>10000</timeout>
    <encoding>utf-8</encoding>
    <type>rest</type> <!-- rest, normal(default)-->
</HTTP_REST>
Network Manager Settings <path>
  • Enter the full package name of the HttpNetworkManager class for native communication control.
  • The HttpNetworkManager describes the development of native communication extensions for data communication.
Contact address <address>
  • Set the default connection address.
Timeout <timeout>
  • Sets the maximum timeout time for network connection attempts.
  • The setting for each text can be set separately as an option value when calling a function.
Encoding <encoding>
  • Set the communication data encoding method. (Default : UTF-8)
Communication type <type>
  • Communication type
    1. rest : Rest Rule data communication
    2. normal : Common servlet (interface.do) rule data communication

socket setting <socket>

  • Create a network alias by giving it the socket server name.
  • Example Code
    <AUTH>
    <path>m.client.app.samples.network.HttpDataNetworkManager</path>
    <address>http://218.232.105.175:8080/api/interface</address>
    <port>8080</port>
    <timeout>10000</timeout>
    <encoding>utf-8</encoding>
</AUTH>
Network Manager Settings <path>
  • Enter the full package name of the SocketNetworkManager class for native communication control.
  • SocketNetworkManager describes the development of native communication extensions for data communication.
Contact address <address>
  • Set the default connection address.
Connection port <port>
  • Set the connection port.
Timeout <timeout>
  • Sets the maximum timeout time for network connection attempts.
  • The setting for each text can be set separately as an option value when calling a function.
Encoding <encoding>
  • Set the communication data encoding method. (Default : UTF-8)

AppManifest configuration example

  • Below is the complete configuration of the AppManifest.xml document. 
<?xml version="1.0" encoding="utf-8"?>
<manifest>
    <!-- log setting -->
    <log>y</log>
 
    <!-- language setting -->
    <lang>ko</lang>
 
    <!-- resource setting -->
    <resource>
        <base_version>100001</base_version> <!-- base resource version -->
        <target>app</target>  <!-- use resource target / app, doc, sdcard -->
        <update>
            <!-- resource update check transaction info -->
            <server>HTTP_REST</server> <!-- resource update target server  -->
            <trcode>/api/res/auto-update</trcode> <!-- /rest/api/res/auto-update(rest) , rsc.auto_update(normal) -->
            <mode>real</mode> <!-- resource update mode / dev, real -->
        </update>
    </resource>
 
    <!-- start page setting -->
    <startpage>
        <name>index.html</name>
        <orient>port</orient>
    </startpage>
 
    <!-- navigation setting -->
    <default_action>
        <orient>default</orient> <!-- default orientation -->
        <animate>default</animate> <!-- default animation -->
        <indicator>y</indicator> <!-- show page switching indicator -->
    </default_action>
 
    <!-- library setting -->
    <library>
        <ext> m.client.app.implementation.ExtendWNInterface </ext>
        <addons>
            <path>m.client.library.addon.db.WNInterfaceDB</path>
            <path>m.client.library.addon.file.WNInterfaceFile</path>
            <path>m.client.library.addon.media.WNInterfaceMedia</path>
            <path>m.client.library.addon.net.WNInterfaceNet</path>
            <path>m.client.library.addon.netext.WNInterfaceNetExt</path>
            <path>m.client.library.addon.popup.WNInterfacePopup</path>
            <path>m.client.library.addon.zip.WNInterfaceZip</path>
        </addons>
        <plugins>
            <path>com.google.zxing.client.android.WNInterfaceQR</path>
        </plugins>
    </library>
 
    <!-- network setting -->
    <network>
        <http>
            <HTTP_REST>
                <path>m.client.app.samples.network.HttpDataNetworkManager</path>
                <address>http://61.74.156.194:83/rest/</address>
                <timeout>10000</timeout>
                <encoding>utf-8</encoding>
                <type>rest</type> <!-- rest, normal(default)-->
            </HTTP_REST>
            <HTTP_NORMAL>
                <path>m.client.app.samples.network.HttpDataNetworkManager</path>
                <address>http://61.74.156.194:83/interface.do</address>
                <timeout>10000</timeout>
                <encoding>utf-8</encoding>
                <type>normal</type> <!-- rest, normal(default)-->
            </HTTP_NORMAL>
            <PUSH>
                <path>m.client.app.samples.network.HttpDataNetworkManager</path>
                <address>http://61.74.156.194:83/api/interface.do</address>
                <timeout>10000</timeout>
                <encoding>utf-8</encoding>
            </PUSH>
        </http>
        <socket>
            <AUTH>
                <path>m.client.app.samples.network.HttpDataNetworkManager</path>
                <address>http://218.232.105.175:8080/api/interface</address>
                <port>8080</port>
                <timeout>10000</timeout>
                <encoding>utf-8</encoding>
            </AUTH>
        </socket>
    </network>
</manifest>

Hardware Acceleration

  • Use Android hardware acceleration.
  • default : software, (hardware : core lib, app : AndroidManifest.xml )
  • Apply information to AndroidManifest.xml for hardware acceleration
    <application android:hardwareAcclerated=”software”>
	<activity …./>
	<activity android:hardwareAcclerated=”app”/>
</application>

Theme

  • Enable Android Theme.
  • default : n, (n: Follow MCore settings, y : Follow AndroidManifest settings)
  • Applying information to AndoridManifest for use with Android Theme
    <application android:theme=”@style/AppTheme” >
	<activity …./>
	<activity android:theme=”@android:style/Theme.Dialog” />
</application>
