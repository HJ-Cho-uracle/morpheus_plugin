개요

  1. 어플리케이션 기본 환경 설정을 하기 위한 Manifest.xml을 제공한다.
  2. 플랫폼에서는 어플리케이션 실행시 자동적으로 Manifest.xml의 정보를 읽어 사용하게 된다.
  3. Manifest.xml의 위치 : 리소스(res) 폴더 구조 참조



앱 및 리소스 배포시 주의 사항

  1. Manifest.xml은 앱내부(asset(Android) 이나 bundle(iOS))의 파일이 사용되므로,
    리소스 배포 파일에 포함되지 않도록 주의 한다\n
  2. <resource><target> 을 'doc'로 설정한다.
  3. <resource><update><mode> 를 'real'로 설정한다.
  4. <log> 를 'n'으로 설정한다.



<log>\n

  • logging 여부를 설정\n
  • enable : 'y' / disable : 'n'

<lang>

  • App의 언어를 설정\n 'system'으로 설정된 경우 디바이스에 설정된 언어를 사용\n 언어-국가별 폴더 이름 정의\n 언어는 ISO 639, BCP 47 을 준수\n 국가는 ISO 3166에 정의

리소스 <resource>

  • 웹 리소스 관련된 설정을 할 수 있다.
  • 예제 코드

								>

						app
						
							</target
								>
							
						  
						<!-- use resource target / app, doc, sdcard -->

						
							<update
								>

							<server
								>
							
						HTTP_REST
						
							</server

								>

							<mode

						<!-- resource update mode / dev, real -->
						
							</update

								>

<base_version>\n

  • 마켓에 올라가는 리소스의 버전\n 사용자가 마켓에서 앱을 다운로드 후 실행\n 기본 버전 ~ 최신 버전까지 업데이트 가능\n 예제 코드\n <!-- 리소스 기본 버전 -->\n <base_version>\n 100001\n </base_version>\n

<target>\n

  • 웹 리소스 사용 폴더를 설정\n
  • 타입\n app : 앱 안에 포함된 리소스를 사용\n (개발 및 리소스 업데이트를 사용하지 않는 경우)\n doc : 앱에 할당된 공간에 있는 리소스를 사용\n (리소스 업데이트와 리소스 복사 기능을 이용하여 배포하는 경우)\n
  • 예제 코드\n <!-- 리소스 타겟 -->\n <target>\n app\n </target>

리소스 업데이트 <update>

  • 리소스 업데이트관련 설정을 한다.
<server>\n
  • <network> > <http>안에 설정된 서버 타겟으로 \n 리소스 업데이트 기능이 있는 게이트웨이 서버이름을 설정\n
  • 예제 코드\n <!-- 게이트웨이 서버 타겟 이름 -->\n <server>\n HTTP_REST\n </server>\n
<trcode>\n
  • 리소스 업데이트 서비스 코드를 설정\n
  • 업데이트 서버가 서비스를 지원하는 형태(rest or interface.do)에 따라 구분\n
  • rest 방식\n '/api/res/auto-update'\n interface.do 방식\n 'rsc.auto_update'\n
  • 예제 코드\n <!-- 리소스 업데이트 서비스 코드 -->\n <trcode>\n /api/res/auto-update\n </trcode>\n
<mode>\n
  • 리소스 업데이트 서비스 모드\n
  • 업데이트 서버의 '개발배포(dev)', '운영배포(real)' 중 요청할 것을 선택\n
  • 개발배포 : dev / 운영배포 : real\n
  • 예제 코드\n <!-- 리소스 업데이트 서비스 코드 -->\n <mode>\n real\n </mode>\n

최초 실행 페이지 <startpage>

  • 최초 실행 페이지를 설정한다.

<name>\n

  • 시작 페이지를 설정\n
  • 최초 실행 페이지에서 반드시 리소스 업데이트를 수행\n

<orient>\n

  • 시작 화면의 orient를 설정\n
  • orient mode의 종류\n land : 가로 모드\n port : 세로 모드\n all : 가로,세로 모드\n default : 시스템에서 설정한 모드를 적용\n

<orient>\n

    orient mode의 종류\n land : 가로 모드\n port : 세로 모드\n all : 가로,세로 모드\n default : 시스템에서 설정한 모드를 적용\n

<default_action>

  • 전화면에 적용되는 orient, animation, indicator의 default 모드를 설정한다.
  • 예제 코드

						default animation 
						-->

						<indicator
						>y

						</indicator
						> 

						<!-- show page switching indicator 
						-->

						</default_action
						>

<orient>\n

  • 전화면에 적용되는 default orient를 설정\n
    • orient mode의 종류\n land : 가로 모드\n port : 세로 모드\n all : 가로,세로 모드\n default : 시스템에서 설정한 모드를 적용\n

<animate>\n

  • 전 화면에 적용되는 default 화면간 이동 animate를 설정\n
  • animate mode의 종류\n default : 시스템 기본 애니메이션\n none : 애니메이션 없음\n slide_left : 왼쪽으로 PUSH\n slide_right : 오늘쪽으로 PUSH\n slide_top : 위쪽으로 PUSH\n slide_bottom : 아래쪽으로 PUSH\n zoom_in : 줌인\n zoom_out : 줌아웃\n fade : 페이드\n modal_left : 왼쪽으로 오버랩\n modal_right : 오른쪽으로 오버랩\n modal_up : 위쪽으로 오버랩\n modal_down : 아래쪽으로 오버랩\n

전화면 이동시 indicator viewing 여부 설정 <indicator>

  • 전화면 이동시 indicator 출력 여부를 설정한다.
  • enable : 'y' / disable : 'n'

확장 인터페이스 <ext>

  • 네이티브 확장 인터페이스 클래스의 풀 패키지명을 입력한다.

에드온 <addons>

  • 자주 사용되는 함수들을 기본으로 제공하는 Addon들 이다.
  • 특별한 상황이 아니면 수정하지 않는것을 권장함\n

플러그인 <plugins>

  • 플러그인으로 제공하는 플러그인 라이브러리들의 확장 인터페이스 클래스 풀 패키지명을 입력한다.
  • 풀 패키지명은 플러그인의 설명 파일에 기제 되어있음

네트워크 설정 : <network>

  • http 및 socket 네트워크를 설정한다.

http 설정 <http>

  • 네트워크 별명으로 http 서버이름을 부여하여 만든다.
  • 예제 코드

								>
							
						 
						
							<path
								>
							
						m.client.app.samples.network.HttpDataNetworkManager
						
							</path
								>
							
						
						
							<address
								>
							
						http://61.74.156.194:83/rest/
						
							</address
								>
							
						
						
							<timeout
								>
							
						10000
						
							</timeout
								>
							
						
						
							<encoding
								>
							
						utf-8
						
							</encoding
								>
							
						
						
							<type
								>
							
						rest
						
							</type
								>
							
						 
						<!-- rest, normal(default)-->
						
							</HTTP_REST
								>
네트워크 메니저 설정 <path>
  • 네이티브 통신 제어를 위한 HttpNetworkManager 클래스의 풀 패키지 명을 입력한다.
  • HttpNetworkManager에는 데이터 통신을 위한 네이티브 통신 확장 개발을 기술 한다.
접속 주소 <address>
  • 기본 접속 주소를 설정한다.
타임아웃 <timeout>
  • 네트워크 연결 시도 최대 타임아웃 시간을 설정한다.
  • 각 전문마다 설정은 함수 호출시 option 값으로 따로 설정이 가능하다.
인코딩 <encoding>
  • 통신 데이터 인코딩 방식을 설정한다\n(Default : UTF-8)
통신 방식 타입 <type>
  • 통신 방식 타입
    1. rest : rest 규칙 데이터 통신
    2. normal : 일반 servlet(interface.do) 규칙 데이터 통신

  • SocketNetworkManager에는 데이터 통신을 위한 네이티브 통신 확장 개발을 기술 한다.
접속 주소 <address>
  • 기본 접속 주소를 설정한다.
접속 포트 <port>
  • 접속 포트를 설정한다.
타임아웃 <timeout>
  • 네트워크 연결 시도 최대 타임아웃 시간을 설정한다.
  • 각 전문마다 설정은 함수 호출시 option 값으로 따로 설정이 가능하다.
